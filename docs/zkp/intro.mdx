对于零知识证明（ZKP），有一个很通俗易懂的例子：「寻找波塞冬小人」，假如在一幅巨型画作（类似清明上河图）中存在许许多多的人物形象与色彩斑斓的背景，Alice 如何找到某个小人在画面中的具体位置呢？这个小人的独立画像可以让 Alice 知道，但 Alice 用尽力气去观察也没有找到，Alice 开始质疑画面上根本就不存在这个小人画像。为了消除 Alice 的疑虑，Bob 用一张面积很大、中间有一个和小人形象一致的破洞的黑布，罩住画作，并挪移破洞到小人的位置，让 Alice 知晓画面上确实存在这么一个小人，但 Bob 并没有暴露小人的具体位置，这就是零知识证明--无需透露重要信息，却能证明重要信息的存在或归属。

其实对于程序员来说，有一个更简单的例子，可以解释说明什么是 ZKP：在 MySQL 数据库系统中执行 

> Select name From student Where name = "Alice";
> 结果返回 
> "Alice"

那么这个 SQL 语句就证明了 student 表中有一个名叫 Alice 的学生，但并没有暴露其他学生的任何信息，以及 Alice 与其他学生之间的关联对比（例如：身高排序的名次）。这个 SQL 语句完成一种证明任务，却没有暴露任何额外的信息，就是一种零知识证明（ZKP）。
